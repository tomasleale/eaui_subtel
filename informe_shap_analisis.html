2. Metodología del Análisis

2.1 ¿Qué es SHAP?

SHAP (SHapley Additive exPlanations) es un método teoricamente fundamentado en teoría de juegos que proporciona explicaciones locales y globales de predicciones de modelos de machine learning. A diferencia del feature importance tradicional:

Feature Importance Tradicional

  • Basado en ganancia media de impureza
  • Valor global único por feature
  • No muestra direccionalidad del impacto
  • Puede estar sesgado en modelos complejos

SHAP

  • Basado en teoría de coaliciones de juegos
  • Valor local POR PREDICCIÓN
  • Muestra si empuja predicción hacia arriba (+) o abajo (-)
  • Fundamentación teórica más sólida

5.2 Interpretación de Gráfico Beeswarm SHAP

El gráfico SHAP Beeswarm Plot muestra cada punto como una observación individual del conjunto de prueba. Las dimensiones son:

  • Eje X (horizontal): Valor SHAP - magnitud y dirección del impacto en la predicción
    • Valores negativos (-): "empujan" la predicción hacia clases más bajas
    • Valores positivos (+): "empujan" hacia clases más altas
  • Eje Y (vertical): Features ordenadas por importancia media
  • Color del punto: Valor del feature en esa muestra
    • Rojo: valor alto del feature
    • Azul: valor bajo del feature
  • Ancho de la nube: Variabilidad del impacto entre muestras
📌 Clave para leer Beeswarm:
  • Si ves puntos rojos a la DERECHA → valores altos de ese feature aumentan la predicción
  • Si ves puntos rojos a la IZQUIERDA → valores altos disminuyen la predicción
  • Una nube dispersa = mucha heterogeneidad en el impacto
  • Una nube concentrada = impacto consistente

5.3 Gráfico BAR - Importancia Media Global

El gráfico BAR muestra la media absoluta de valores SHAP por feature. Representa importancia global sin dirección.

  • Barras más largas = feature que más impacta las predicciones en promedio
  • Barras cortas = feature con impacto menor y más acotado
  • Útil para identificar top-K features a nivel global

7.2 Limitaciones del Modelo

  • Accuracy 27.5%: Aunque bajo en valor absoluto, es razonable para un problema de 6 clases (baseline: 26.3% si predice mayoritaria C3).
    • GSE es multidimensional y depende de ingresos, educación, ocupación - no solo acceso a internet
    • Nuestros features son parciales (acceso/uso) - faltan educación, ocupación, ingresos
  • Colinealidad: Features como intensidad_uso derivan de frecuencia_internet y tiempo_diario
  • Causalidad: SHAP muestra asociación, no causalidad. Edad puede ser proxy de otros factores no observados

7.3 Mejoras Futuras

  1. Incluir features de educación (nivel alcanzado), ocupación, y ingresos del jefe de hogar
  2. Incorporar ubicación geográfica (región, zona) con mayor detalle
  3. Probar otros modelos (GradientBoosting, XGBoost) y comparar SHAP
  4. Realizar análisis de interacción SHAP (ej: edad × zona)
